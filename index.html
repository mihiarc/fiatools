Four tools, one ecosystem

Each tool handles a specific domain while sharing common patterns and seamless interoperability.

pyFIA

Data Foundation

High-performance Python API for FIA survey data. DuckDB backend, Polars DataFrames, EVALIDator-compatible statistics.

  • Trees per acre, biomass, volume
  • Post-stratified estimation
  • 10-100x faster queries
View repository →

gridFIA

Spatial Analysis

Zarr-based storage for BIGMAP raster data. 327 species at 30m resolution, optimized for localized biomass analysis.

  • Species diversity indices
  • Any US state or county
  • Cloud-optimized Zarr
View repository →

pyFVS

Growth Simulation

Python implementation of FVS Southern variant. Individual tree growth models for loblolly, shortleaf, longleaf, and slash pine.

  • Chapman-Richards growth curves
  • Yield table generation
  • Crown competition modeling
View repository →

askFIA

AI Interface

Conversational interface for forest data. Natural language queries powered by the FIAtools ecosystem.

  • Natural language queries
  • Automated analysis
  • Report generation
View repository →

Quick start

Get running in minutes with pip install and a few lines of code.

from pyfia import FIA, biomass, tpa, volume

# Connect to FIA database
with FIA("path/to/FIA_database.duckdb") as db:
    # Filter to North Carolina
    db.clip_by_state(37)
    db.clip_most_recent(eval_type="EXPVOL")
    
    # Get forest metrics
    tpa_results = tpa(db, tree_domain="STATUSCD == 1")
    biomass_results = biomass(db, by_species=True)
    volume_results = volume(db, land_type="forest")
