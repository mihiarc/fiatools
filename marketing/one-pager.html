The Challenge

Working with USDA Forest Service FIA data requires navigating complex database schemas, understanding statistical estimation procedures, and integrating spatial and tabular data sources. Existing tools like EVALIDator provide web interfaces but lack programmatic access for reproducible research workflows.

The Solution

FIAtools provides a unified Python ecosystem that wraps FIA data in modern, high-performance APIs. From survey plot analysis to spatial biomass mapping to growth simulation—all with consistent interfaces, proper statistical methods, and 10-100× faster performance than existing alternatives.

Four Tools, One Ecosystem

pyFIA

Data Foundation

High-performance API for FIA survey data. DuckDB + Polars backend. EVALIDator-compatible statistics.

gridFIA

Spatial Analysis

BIGMAP raster processing. Zarr-based storage for 327 species at 30m resolution. Diversity indices.

pyFVS

Growth Modeling

Python FVS (Southern variant). Individual tree growth for loblolly, shortleaf, longleaf, slash pine.

askFIA

AI Interface

Natural language queries. Conversational access to the entire FIAtools ecosystem.

Key Features

  • Exact statistical compatibility with EVALIDator
  • Post-stratified estimation with proper variance
  • DuckDB columnar storage for fast queries
  • Cloud-optimized Zarr for spatial data
  • EVALID-based temporal filtering
  • Chapman-Richards growth equations
  • MIT licensed, open source
  • Comprehensive documentation
